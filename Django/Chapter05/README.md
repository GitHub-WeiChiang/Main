Chapter05 網址的對應與委派
=====
* ### URLconf 簡介
    * ### Django 2.0 後，使用 path() 進行網址委派 (2.0 以前是使用 re_path())。
    * ### settings.py 中的 ROOT_URLCONF 指定了 urlpatterns 所在的模組。
* ### Django 2.0 前的 re_path()
    * ### 在字串前面的 "r" 代表 Python 解譯器要保持後面字串的原貌，不可以處理跳脫字元，用於 RE 解譯的字串都會加上它。
    * ### "^" 表示定義開頭字串。
    * ### "$" 表示定義結尾字串。
    * ### 對於首頁無需加上 "/" 字元，也就是應該呈現為 "^$"，而非 "^/$"。
